# StyleHub — веб-версия интернет-магазина одежды

## Описание проекта
**StyleHub** — это учебное веб-приложение интернет-магазина одежды, разработанное в рамках курсового проекта по дисциплине **«Проектирование и обеспечение бесперебойной работы web-сайта»**.

Проект реализован в формате **SPA (Single Page Application)** и предназначен для демонстрации основных возможностей современного интернет-магазина:
- просмотр каталога товаров;
- переход к карточке товара;
- добавление товаров в корзину;
- добавление товаров в избранное;
- регистрация и вход пользователя;
- доступ к личному кабинету.

## Цель проекта
Разработать удобное веб-приложение интернет-магазина одежды с современным интерфейсом, понятной навигацией и базовым функционалом электронной коммерции.

## Основной функционал
В приложении реализованы следующие возможности:

- главная страница магазина;
- каталог товаров;
- карточка отдельного товара;
- поиск, фильтрация и сортировка;
- корзина покупок;
- избранные товары;
- регистрация пользователя;
- вход в систему;
- личный кабинет;
- сохранение данных корзины и избранного в `localStorage`;
- хранение пользователей в базе данных SQLite;
- хеширование паролей на серверной стороне.

## Архитектура проекта
Проект разделён на две основные части:

### 1. Клиентская часть
Клиентская часть отвечает за пользовательский интерфейс и взаимодействие с пользователем.  
Реализована на React с использованием компонентного подхода.

Основные страницы:
- Home
- Catalog
- Product Page
- Cart
- Favorites
- Login
- Register
- Profile
- About

### 2. Серверная часть
Серверная часть отвечает за регистрацию и авторизацию пользователей.  
Для хранения данных используется база данных **SQLite**.

Сервер реализует:
- регистрацию пользователя;
- проверку существования email;
- вход в систему;
- сравнение паролей через `bcryptjs`;
- возврат данных пользователя после успешной авторизации.

## Хранение данных
В проекте используется два способа хранения данных:

### SQLite
Используется для хранения зарегистрированных пользователей:
- `id`
- `name`
- `email`
- `password` (в хешированном виде)

### localStorage
Используется для хранения:
- текущего пользователя;
- корзины;
- избранных товаров.

Такое решение позволяет не менять основную логику проекта и сохранить уже реализованный интерфейс.
